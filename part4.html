• Survivors tend to be slightly younger (mean age ~28.3) vs non-survivors (~30.6), but the effect is moderate.

Fare shows the highest positive correlation with survival (+0.26), while Pclass shows the highest negative correlation (−0.34, noting that higher class number = lower class). Sex_encoded (female=1) shows very strong positive correlation (+0.54). Age shows a weak negative correlation (−0.07).

6. Preprocessing Recommendations

🔧 Missing Data Strategy

  • Age (19.9%): Impute using median age per Sex+Pclass group (more accurate than global median)
  • Cabin (77.1%): Don't impute — create binary Has_Cabin feature (cabin known = likely wealthier)
  • Embarked (0.2%): Impute with mode 'S' (Southampton)

⚙️ Feature Engineering

  • Extract Title from Name (Mr, Mrs, Miss, Master, Rare) — proxy for age/social status
  • Create FamilySize = SibSp + Parch + 1
  • Create IsAlone binary flag (FamilySize == 1)
  • Apply log(Fare + 1) to normalize skewed distribution
  • Bin Age into groups: Child, Teen, Adult, Senior
  • Extract Deck letter from Cabin where available

📊 Encoding

  • Sex: Binary encode (female=1, male=0)
  • Embarked: One-hot encode (3 categories)
  • Pclass: Treat as ordinal or one-hot
  • Title: Ordinal or one-hot after grouping rare titles

🚫 Drop / Cautious Features

  • PassengerId: Drop — purely sequential ID
  • Name: Drop after extracting Title
  • Ticket: Drop (high cardinality; limited signal without complex NLP)
  • Fare: Watch for multicollinearity with Pclass
Top Modeling Signals (ranked): Sex > Pclass > Fare > Title > FamilySize > Age > Embarked. A gradient boosted tree (XGBoost/LightGBM) or Random Forest will likely perform well given the mix of categorical and numerical features with moderate non-linear interactions.
